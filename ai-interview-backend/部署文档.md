# AI-Interview-Agent（智面 · Agent 版）— 部署文档

> 适用版本：`ai-interview-agent` 后端（FastAPI + PG/pgvector + Redis + Celery + LangChain + DeepSeek + DashScope）
> 本文档对应代码库：`ai-interview-backend/`

---

## 写在前面

后端方面
1. 自行安装好 VMware 虚拟机和 Ubuntu 系统（推荐 18.04+，**20.04/22.04 LTS 更佳**）
2. VMware 网络设置：编辑 → 虚拟网络编辑器 → NAT 模式，确保 DHCP 分配正常
3. Docker 需要联网拉取基础镜像（pgvector/pgvector、redis），国内需配置镜像加速
4. 容器内存建议 **≥ 4 GB**（LangChain + DashScope + pgvector 跑起来后内存占用上升）
5. 创建好数据表后，建议检查是否成功创建（命令见下方）
6. 本项目在原 `ai-interview` 基础上**新增了**：
   - **题库 RAG**（pgvector 三分支召回）
   - **知识库 RAG**（pgvector 文档切片 + 向量化）
   - **岗位匹配 Agent**（LangChain Tool Calling Agent）
   对应的 `.env` 必须配置 **DeepSeek** + **DashScope** 两组 Key，否则 AI 闭环跑不通。

## 项目简介

基于大模型的 **AI 模拟面试系统**。相比上一版，**新增 RAG 题库 + RAG 知识库 + 岗位匹配 Agent**，构建完整的"**简历解析 → 岗位匹配 → RAG 出题 → AI 评分 → 面试报告**"求职闭环：

- 用户上传 PDF 简历，系统先调用 AI 解析简历并落库
- 基于 **岗位匹配 Agent**（LangChain Tool Calling Agent）对候选人画像进行分析，推荐更适合的岗位
- 进入目标岗位的专项模拟面试，**题库 RAG** 召回高相关题目，AI 评分时引入 **知识库 RAG** 作为参考
- 输出结构化面试报告，帮助用户明确能力差距和下一步学习方向

### 功能特性

- 📄 简历上传与 AI 智能解析（PDF → 结构化数据）
- 🤖 **岗位匹配 Agent**（5 个工具串联：`get_parsed_resume` → `build_candidate_profile` → `match_positions` → `get_position_interview_focus` → `start_mock_interview`）
- 📚 **题库 RAG**（pgvector 三分支召回：题库充分时 AI 选题；题库不足时 AI 兜底补全；题库为空时纯 AI 生成）
- 📖 **知识库 RAG**（文档切片 + DashScope Embedding + Celery 异步向量化 + 评分/答疑时召回）
- 🎤 AI 模拟面试（多轮对话，逐题问答）
- 📊 AI 评分（注入题库的 `reference_answer` + `key_points` + 知识库检索结果）
- 📝 流式评分（SSE，逐字推送）
- 📈 面试报告生成
- 🛠 后台管理系统（用户管理、面试记录查看、**题库管理**、**知识库管理**、**岗位模板维护**）

### 项目亮点

**1. 架构设计**
- Router → Service → Model 三层架构，职责分离
- 双端 API 设计：Client API（用户端）和 Backoffice API（管理端）共享 Service 层
- `router_registry.py` 统一注册所有路由，新增模块只需加 `RouteConfig`（参见 [app/route/router_registry.py](app/route/router_registry.py)）
- 统一响应格式（ApiResponse）+ 分层异常体系

**2. RAG 双链路**
- 题库 RAG：结构化题库向量化存储，召回时携带 `bank_id / reference_answer / key_points` 一路传到评分阶段，保证评分依据可追溯
- 知识库 RAG：管理端上传文档 → Celery 异步切片 → DashScope Embedding → 写 pgvector；评分时按 `KNOWLEDGE_TOP_K` 召回
- 题库召回参数（`QUESTION_BANK_MIN_SCORE / RECALL_FACTOR / TOP_K`）可在 `.env` 调优，调整后改 `.env` + `docker-compose restart app celery-worker` 即生效

**3. Agent 工程化**
- LangChain `create_tool_calling_agent` + `AgentExecutor`
- 工具 docstring 即 LLM 的 schema，改 docstring 等于改提示词
- 候选人画像 → 岗位模板匹配 → 面试重点 → 一键开面试，全链路工具调用

**4. AI 能力统一封装**
- `app/services/client/ai_service.py` 是项目里的 **AI 能力中枢**，统一封装：
  - `parse_resume` / `analyze_resume`
  - `generate_questions` / `generate_with_seeds` / `select_and_adapt_questions`
  - `evaluate_answer`（流式）
  - `generate_report`
  - 题库补题与微调
- 业务层禁止直接 `import openai`，所有 DeepSeek 调用走 `AsyncOpenAI`（OpenAI 兼容 SDK）

**5. 性能与工程化**
- 全链路异步：FastAPI + SQLAlchemy async + asyncpg + AsyncOpenAI
- 数据库连接池优化：`pool_size=20`、`pool_pre_ping`、`pool_recycle=1800s`
- pgvector 用 `cosine_distance` + `hnsw vector_cosine_ops` 索引（迁移自动建）
- Celery + Redis 异步任务解耦：知识库向量化、邮件发送等耗时操作不阻塞主请求
- JWT 无状态认证，Access Token 验证无需查库
- AI 对话历史截断（保留最近 6 条），控制 Token 用量

## 第一步：VM 环境准备

### 1.1 安装 SSH 服务（方便远程操作）

```bash
sudo apt-get update
sudo apt-get install -y openssh-server curl
sudo systemctl start ssh
sudo systemctl enable ssh
```

> 安装完成后，可以在 Windows 上通过终端 SSH 连接 VM，方便复制粘贴命令
建议使用编辑器打开项目，然后在编辑器里的终端通过 SSH 连接 VM，方便复制粘贴命令：

> ```bash
> ssh 用户名@VM的IP地址
> ```

### 1.2 安装 VMware Tools（共享文件夹必需）

```bash
sudo apt-get install -y open-vm-tools open-vm-tools-desktop
sudo reboot
```

> 重启后 VMware Tools 生效，才能使用共享文件夹功能。

### 1.3 配置 VMware 共享文件夹

1. VMware → 虚拟机设置 → 选项 → 共享文件夹 → 添加后端项目目录（`ai-interview-backend`）
2. 开启 VM，挂载共享文件夹：

```bash
sudo mkdir -p /mnt/hgfs
sudo vmhgfs-fuse .host:/ /mnt/hgfs -o allow_other
```

> ⚠️ 每次 VM 开机都需要执行挂载命令，否则看不到共享文件夹，项目也就无法启动

3. 验证：

```bash
ls /mnt/hgfs/
# 应该能看到共享的 ai-interview-backend 文件夹
```

### 1.4 安装 Docker

```bash
sudo apt-get update
sudo apt-get install -y docker.io
sudo systemctl start docker
sudo systemctl enable docker
sudo usermod -aG docker $USER
```

> 执行完 `usermod` 后需要重新登录 VM（退出 SSH 再重连）才能免 sudo 使用 docker。
> **⚠️ 内存设置**：VMware → 虚拟机设置 → 内存，**至少分配 4 GB**，否则 LangChain + Embedding 跑起来会被 OOM kill。

### 1.5 安装 Docker Compose V2

Ubuntu 18.04 的 apt 源中 docker-compose 版本太旧（1.17），不支持本项目的配置语法。需要手动安装新版。

**安装方法：从项目中复制（作者已提供二进制文件）**

项目目录中已包含 `docker-compose-linux-x86_64` 文件（60 MB 的 docker-compose v2.24.0 离线可执行文件），直接复制即可：

```bash
sudo cp /mnt/hgfs/ai-interview-backend/docker-compose-linux-x86_64 /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
```

> 如果系统中已有旧版 docker-compose，先卸载：`sudo apt-get remove -y docker-compose`，然后执行 `hash -r` 刷新命令缓存。

```bash
docker-compose --version
# 应显示 Docker Compose version v2.24.0
```

### 1.6 配置 Docker 镜像加速（必须，否则无法拉取镜像）

国内网络无法直接访问 Docker Hub，必须配置镜像加速：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": [
    "https://docker.1ms.run",
    "https://docker.xuanyuan.me",
    "https://docker.m.daocloud.io"
  ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

### 1.7 解决 Docker 凭证存储问题（SSH 远程操作时可能遇到）

如果执行 docker 命令时报错 `Cannot autolaunch D-Bus without X11 $DISPLAY`，执行：

```bash
sudo apt-get remove -y golang-docker-credential-helpers
```

### 1.8 设置 VM 静态 IP（推荐）

固定 IP 后前端 `vite.config.js` 不用反复改。

先查看当前 IP 和网卡名：

```bash
ip addr show
```

**如果系统使用 NetworkManager（Ubuntu 桌面版常见）：**

```bash
# 查看连接名称
nmcli con show

# 设置静态 IP（根据实际网段和连接名修改）
sudo nmcli con mod "有线连接 1" ipv4.method manual ipv4.addresses 你的IP/24 ipv4.gateway 网关IP ipv4.dns "8.8.8.8,114.114.114.114"
sudo nmcli con up "有线连接 1"
```

**如果系统使用 netplan（Ubuntu Server 常见）：**

```bash
sudo nano /etc/netplan/01-netcfg.yaml
```

```yaml
network:
  version: 2
  ethernets:
    ens33:
      dhcp4: no
      addresses:
        - 你的IP/24
      gateway4: 网关IP
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]
```

```bash
sudo netplan apply
```

> 网关地址一般是网段的 `.2`（VMware NAT 模式默认），可在 VMware 虚拟网络编辑器中确认。

### 1.9 解决 apt-get update 报错（如遇到）

如果 `sudo apt-get update` 报 `appstreamcli` 相关错误（GLib-CRITICAL、core dumped），执行：

```bash
sudo rm /etc/apt/apt.conf.d/50appstream
sudo apt-get update
```

## 第二步：后端部署（Docker Compose 路线，推荐）

### 2.1 配置环境变量

```bash
cd /mnt/hgfs/ai-interview-backend

# 复制环境配置文件
cp .env.example .env
```

> 因为使用了 VMware 共享文件夹，`.env` 文件在 Windows 和 VM 之间是实时同步的。
> 所以可以直接在 Windows 上用编辑器（记事本、VS Code 等）打开 `.env` 进行编辑，保存后 VM 中立即生效，比在终端里用 nano 方便得多。

#### 2.1.1 必填项（少了跑不通 AI 闭环）

| 配置项 | 说明 |
|--------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API Key，[申请地址](https://platform.deepseek.com/) |
| **`DASHSCOPE_API_KEY`** | 阿里云 DashScope API Key，[申请地址](https://dashscope.console.aliyun.com/)，开通"通用文本向量"服务。**题库和知识库的 Embedding 都靠它**，不填则向量化全部失败 |

#### 2.1.2 RAG 参数（按需调整，默认即可）

| 配置项 | 默认值 | 用途 | 调整建议 |
|--------|--------|------|----------|
| `KNOWLEDGE_EMBEDDING_MODEL` | `text-embedding-v3` | Embedding 模型 | 改了要同步重建索引 |
| `KNOWLEDGE_EMBEDDING_DIM` | `1024` | 向量维度 | **强约束**：与 PG 列 `Vector(1024)` 硬绑定，改了必须 `ALTER TABLE ... TYPE vector(N)` + 跑全量重建任务 |
| `KNOWLEDGE_CHUNK_SIZE` | `500` | 文档切分大小 | 视文档长度调整 |
| `KNOWLEDGE_CHUNK_OVERLAP` | `50` | 切片重叠 | 一般不改 |
| `KNOWLEDGE_TOP_K` | `4` | 知识库召回条数 | 评分时注入的参考片段数 |
| `KNOWLEDGE_MIN_SCORE` | `0.3` | 知识库召回相似度阈值 | 太低会注入噪声，太高会漏召回 |
| `QUESTION_BANK_MIN_SCORE` | `0.7` | 题库召回相似度阈值 | 调低可放宽命中 |
| `QUESTION_BANK_RECALL_FACTOR` | `2` | 题库召回倍率（实际召回数 = TOP_K × 该因子） | 题库少时可调高 |
| `QUESTION_BANK_TOP_K` | `20` | 题库召回数 | 调小可加速但可能漏题 |

#### 2.1.3 容器网络配置（一般不动）

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `POSTGRES_HOST` | `postgres` | **必须用 docker service 名**，容器内互联。改 `localhost` 会导致 app 容器连不到 db |
| `POSTGRES_PORT` | `5432` | 容器内端口（宿主机映射为 `5434`） |
| `REDIS_HOST` | `redis` | 同上，docker service 名 |
| `REDIS_PORT` | `6379` | 容器内端口（宿主机映射为 `6382`） |

> 如果是 Windows 本地裸跑（不走 docker），需要把 `POSTGRES_HOST` / `REDIS_HOST` 改成 `localhost`。

#### 2.1.4 其他保持默认

JWT、AWS、邮件、BREVO 等配置沿用 `.env.example` 默认即可。

## 日常启动与关闭

部署完成后，每次使用项目只需以下操作：

### 启动项目

**1. VM 端（后端）**

```bash
# 挂载共享文件夹（每次 VM 开机后都要执行）

# 进入项目目录
cd /mnt/hgfs/ai-interview-backend

# 启动所有后端服务
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml up -d

# 确认服务正常
curl http://localhost:8006/api/v1/config/health
```

**2. Windows 端（前端）**

```bash
cd ai-interview-frontend
npm run dev

# 后台管理（端口 3001）— 新开一个终端

npm run dev
```

### 关闭项目

**1. Windows 端：** 在前端终端按 `Ctrl + C` 停止

**2. VM 端：**

```bash
cd /mnt/hgfs/ai-interview-backend
```

> `down` 会停止并移除容器，但数据库数据不会丢失（存在 Docker Volume 中）。
> 如果只是暂时不用，也可以不执行 `down`，容器配置了 `restart: unless-stopped`，VM 重启后会自动恢复。

## 故障排查

### 后端启动失败

```bash
sudo docker logs ai-interview-app --tail 30
# 常见原因：PostgreSQL 还没 ready，等 healthy 后重启
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml restart app
```

### 数据表不存在

```bash
# 重新执行迁移
sudo docker exec -it ai-interview-app alembic upgrade head

# 如果 alembic 显示已在 head 但表不存在，生成新迁移
sudo docker exec -it ai-interview-app alembic revision --autogenerate -m "create tables"
sudo docker exec -it ai-interview-app alembic upgrade head

# 检查表
sudo docker exec -it ai-interview-postgres psql -U demo -d ai_interview -c "\dt"

# 缺 vector 扩展？
sudo docker exec -it ai-interview-postgres psql -U demo -d ai_interview -c "CREATE EXTENSION IF NOT EXISTS vector;"
```

### Vector 类型报错（"type 'vector' does not exist"）

**根因**：PostgreSQL 镜像不是 `pgvector/pgvector:pg16`。

```bash
sudo docker inspect ai-interview-postgres | grep Image
# 应看到："Image": "pgvector/pgvector:pg16"

# 如果不是，删掉旧容器和数据卷，重新启动
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml down -v
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build
```

### Embedding 失败 / DashScope 401

```bash
# 1. 确认 .env 里有 DASHSCOPE_API_KEY
grep DASHSCOPE .env

# 2. 确认容器内能访问 DashScope
sudo docker exec -it ai-interview-app curl -I https://dashscope.aliyuncs.com

# 3. 查看 Celery Worker 详细日志
sudo docker logs ai-interview-celery-worker --tail 100
```

### 知识库上传后未向量化 / 卡在 pending

```bash
# 1. 看 Celery Worker 是否有报错
sudo docker logs ai-interview-celery-worker --tail 50

# 2. 重启 Celery 触发未完成任务
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml restart celery-worker

# 3. 在后台"知识库管理"页手动点"重新向量化"
```

### 题库出题全靠 AI 生成（没有命中题库）

```bash
# 1. 确认题库已向量化
sudo docker exec -it ai-interview-postgres psql -U demo -d ai_interview -c "SELECT COUNT(*) FROM question_bank WHERE embedding IS NOT NULL;"

# 2. 调低召回阈值（题库少时）
# 在 .env 里把 QUESTION_BANK_MIN_SCORE 从 0.7 调到 0.5
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml restart app
```

### 岗位匹配 Agent 报错 / 工具调用失败

```bash
# 1. 确认岗位模板已 seed
sudo docker exec -it ai-interview-postgres psql -U demo -d ai_interview -c "SELECT position_tag, title FROM position_templates;"
# 应返回 10 条左右岗位

# 2. 没数据就重新 seed
sudo docker exec -it ai-interview-app python scripts/seed_position_templates.py
```

### 前端请求后端失败

- 检查 VM 容器是否运行：`sudo docker-compose ps`
- 检查 `vite.config.js` 中的 IP 是否正确
- 检查 VM 防火墙：`sudo ufw status`（如果开启了，放行 8006 端口）
- 确认 `.env` 中 `HTTP_PROXY` 和 `HTTPS_PROXY` 为空

### 共享文件夹看不到

```bash
# 确认 VMware Tools 已安装
dpkg -l | grep open-vm-tools

# 重新挂载
sudo vmhgfs-fuse .host:/ /mnt/hgfs -o allow_other,nonempty
```

### AI 功能报错

- 检查 `.env` 中 `DEEPSEEK_API_KEY` / `DASHSCOPE_API_KEY` 是否正确填写
- 确认 VM 能访问外网：`curl https://api.deepseek.com`
- `.env` 修改后需要 `down` + `up` 重新创建容器（`restart` 不会重新读取 `.env`）

### Docker 拉取镜像超时

- 确认已配置 Docker 镜像加速（见 1.6 节）
- 配置后需要重启 Docker：`sudo systemctl restart docker`

### 重置所有数据

```bash
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml down -v
sudo docker-compose -f docker-compose.yml -f docker-compose.dev.yml up -d --build
# 等待 PostgreSQL 启动后重新迁移
sudo docker exec -it ai-interview-app alembic upgrade head
sudo docker exec -it ai-interview-app python scripts/create_first_admin.py
sudo docker exec -it ai-interview-app python scripts/seed_position_templates.py
```

## 项目红线（修改前必读）

这几条踩坑会直接报错，改之前确认：

| 红线 | 说明 |
|------|------|
| **PG 镜像必须 `pgvector/pgvector:pg16`** | 普通 `postgres` 镜像装不上 `Vector` 类型 |
| **Embedding 维度硬编码 1024** | 写死在 `Vector(1024)` 列，换模型需 `ALTER TABLE ... TYPE vector(N)` + 跑重建任务 |
| **Celery 任务不能 `asyncio.run` 包 async** | 业务用 `embed_text`/`embed_texts`，Celery 用 `embed_text_sync`/`embed_texts_sync` |
| **新 Celery 任务** | 在 `app/schedule/jobs/` 新建后加进 [celery_app.py](app/core/celery_app.py) 的 `include` 列表 |
| **新接口** | 必须在 [router_registry.py](app/route/router_registry.py) 的 `CLIENT_ROUTES` 或 `BACKOFFICE_ROUTES` 注册，否则不生效 |
| **改岗位模板字段** | 同步改 `models/position_template.py` + `scripts/seed_position_templates.py` + `_calculate_match_score`（在 `position_agent_tools.py`）三处 |
| **`POSTGRES_HOST` 在容器里必须是 `postgres`** | 改 `localhost` 会导致容器内 app 连不上 db |
